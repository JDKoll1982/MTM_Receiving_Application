Folder Structure

How to organize skills with scripts, references, assets, and templates

What Each Folder Is For

üìÑ

SKILL.md (Required)

Main instructions and workflows. Keep it under ~500 lines. Link out to references for long content.

üîß

scripts/

Executable automation. Use for deterministic operations like audits, validations, file generation, or report creation.

üìö

references/

Detailed docs Copilot reads on-demand. Use for long workflows, deep explanations, and decision records.

üé®

assets/

Static files used as-is (images, icons, brand templates). Copilot should not edit these.

üìã

templates/

Starter files Copilot will modify (scaffolds for code, config, docs). Put placeholders in templates for easy replacement.

Assets vs Templates (Common Confusion)

Assets are used unchanged. Templates are a starting point that Copilot edits.

Folder Copilot edits it? Best for Examples
assets/ No Static resources Logo PNG, report HTML shell, custom font
templates/ Yes Starter scaffolds ViewModel template, service template, config template

How to Reference Files (So They Load)

To ensure Copilot can find and load your resources, reference them with relative paths from inside SKILL.md:

See detailed rules in [architecture-rules.md](./references/architecture-rules.md)
Run the validator: [validate-mvvm.ps1](./scripts/validate-mvvm.ps1)
Start from template: [viewmodel-template.cs](./templates/viewmodel-template.cs)
Tip: Avoid absolute paths. Keep everything relative to the skill folder.

When to Add a Script

Add a script when you need deterministic behavior that should not rely on natural language generation.

  • Finding MVVM violations at scale
  • Generating a standardized report
  • Bulk updating files safely
  • Validating naming conventions

When to Add References

Add reference docs when your SKILL.md is getting too long or you want deeper explanations:

  • Long workflows (> 5 steps)
  • Architecture rules and guardrails
  • Common violations with examples
  • Troubleshooting guides

When to Add Templates

Add templates when you want repeatable scaffolding that Copilot can customize per feature:

  • New ViewModel scaffolds
  • Service interface + implementation pairs
  • DAO boilerplate with stored procedure calls
  • Test scaffolds (xUnit)

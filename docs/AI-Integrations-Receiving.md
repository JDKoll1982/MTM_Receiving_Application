# AI Integration Opportunities — Module_Receiving

Last Updated: 2026-03-20
Status: Draft — For Review

> All entries are ideas for discussion. Nothing here is committed for implementation.
> Every feature is subject to the privacy rules defined in [AI-Integration-Spec.md](AI-Integration-Spec.md).
> The AI only ever **suggests** — a human always makes the final decision.

## How to Read This Table

| Column                 | Meaning                                                               |
| ---------------------- | --------------------------------------------------------------------- |
| **ID**                 | Short reference code used when discussing or prioritising items       |
| **Workflow Step**      | The screen or moment in the receiving process where this would appear |
| **Feature Name**       | A human-readable label for the AI capability                          |
| **What the AI Does**   | Plain description of what the AI actually produces                    |
| **What the User Sees** | The UI pattern — chip, banner, tooltip, etc.                          |
| **Trigger**            | When does the AI activate — on typing, on save, on load, etc.         |
| **Value Delivered**    | Why this matters to the person doing the receiving                    |
| **Risk / Concern**     | What could go wrong or what needs careful thought                     |
| **Effort Guess**       | Low / Medium / High — rough complexity, not a time estimate           |
| **Tier Needed**        | Minimum GitHub Copilot tier required (Free / Individual / Business)   |

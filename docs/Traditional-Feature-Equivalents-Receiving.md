# Traditional (Non-AI) Feature Equivalents — Module_Receiving

Last Updated: 2026-03-20
Status: Draft — For Reference

> This document is the **companion to** [AI-Integrations-Receiving.md](AI-Integrations-Receiving.md).
> It shows how each of the same 34 features could be built **without any AI** — using pure
> application logic, database queries, configuration rules, and standard UI patterns.
>
> Use this table to compare the cost of the traditional approach against the AI approach,
> and to decide which path is right for a given feature.

---

## How to Read This Table

| Column                       | Meaning                                                                                  |
| ---------------------------- | ---------------------------------------------------------------------------------------- |
| **ID**                       | Same ID as the companion AI table (replace `AI-` with `TR-` for the traditional version) |
| **AI Ref**                   | The corresponding AI-Integration ID for cross-reference                                  |
| **Workflow Step**            | The screen or moment in the receiving process                                            |
| **Feature Name**             | Same human-readable label as the AI version                                              |
| **How It Works Without AI**  | The classical implementation approach — rules, queries, lookups, config tables           |
| **What the User Sees**       | The UI pattern — same goal, different engine                                             |
| **Trigger**                  | When it activates                                                                        |
| **Data Required**            | What data the feature needs to work (tables, config, history)                            |
| **Limitation vs AI version** | Where the traditional approach falls short compared to AI                                |
| **Difficulty**               | Low / Medium / High — how hard to build in this codebase                                 |
